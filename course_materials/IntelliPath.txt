The main idea of this project is to develop a navigation application that integrates real time traffic data and applies machine learning algorithms to learn from historical and live traffic patterns. Unlike traditional navigation systems that only react to current congestion, this app aims to predict traffic conditions and optimize routes proactively.
Project Main Idea

Problem Formulation
Input: Historical and live traffic data
Output: Future road speed/travel time
Task: Supervised regression

Commuting in traffic sucks…
Los Angeles traffic follows strong recurring patterns such as rush hours, school pickup times, and major events. Most navigation systems react to congestion only after it happens, rather than anticipating predictable traffic behavior.
This project aims to transform commuting into a smarter, less stressful experience by developing a navigation system that learns from both historical and live traffic data. By incorporating machine learning, the system can anticipate congestion and adapt route recommendations proactively.
The core objective is to evaluate whether learning-based routing can outperform traditional static or greedy approaches when future traffic uncertainty is considered—ultimately reducing travel time and improving overall efficiency.

The system first predicts future traffic conditions using machine learning.
These predictions become the edge weights in a road network graph.
The A* routing algorithm then computes the best path using predicted travel times rather than current congestion.

Summary
This project explores how statistical traffic prediction combined with reinforcement learning can produce smarter route recommendations that anticipate congestion rather than react to it. By focusing on Los Angeles traffic patterns and individual ETA optimization, the project demonstrates a practical application of machine learning to real-world transportation systems.
